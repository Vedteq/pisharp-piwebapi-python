# Session Log — 2026-03-24

## Research

Searched for PI Web API endpoint coverage, competing Python SDKs, and common user pain points:

- **Official AVEVA PI Web API Reference**: Reviewed controllers at docs.aveva.com
- **Competing SDKs**: PIconnect, pi-web-sdk (PyPI), dcbark01/PI-Web-API-Client-Python, pidevguru client
- **Common gaps in other SDKs**: Inconsistent error handling, missing plot endpoint, no attribute-by-path lookup

## Gaps Found

### Bugs
1. **`eventframes.py` and `servers.py` used `resp.raise_for_status()` instead of SDK's `raise_for_response()`** — bypassing typed exception hierarchy (NotFoundError, AuthenticationError, etc.)

### Missing Endpoints (high-impact)
1. `GET /attributes?path=...` — attribute lookup by full AF path (very common operation)
2. `GET /streams/{webId}/plot` — plot-optimized data retrieval (used by PI Vision)
3. `GET /streams/{webId}/end` — end-of-stream value
4. `GET /streams/{webId}/recorded` with `filterExpression` parameter
5. `GET /streamsets/{webId}/summary` — multi-stream summary in one call
6. `GET /calculations/...` — calculated/expression data
7. `GET /channels/...` — WebSocket streaming (complex, low priority)

### Quality Improvements Needed
- Element templates resource (`/elementtemplates`)
- Attribute templates resource
- Analysis management endpoints
- Notification rules management
- Security identity/mapping endpoints

## Implemented This Session

### 1. Bug Fix: Consistent error handling in eventframes.py and servers.py
- Replaced all `resp.raise_for_status()` with `raise_for_response(resp)` (sync) and `await raise_for_response_async(resp)` (async)
- Added missing import for `raise_for_response` / `raise_for_response_async`
- Added error-path tests (404, 401) for both modules
- Improved docstrings to Google-style with Args/Returns/Raises

### 2. Feature: `get_attribute_by_path` on elements mixin
- `GET /attributes?path=\\AF_SERVER\DB\Element|Attribute`
- Sync + async versions
- Happy-path and error-path (404) tests

### 3. Feature: `get_plot` on streams mixin
- `GET /streams/{webId}/plot` with `startTime`, `endTime`, `intervals` parameters
- Returns `StreamValues` (same shape as recorded/interpolated)
- Sync + async versions
- Happy-path, parameter-forwarding, and error-path (404) tests
