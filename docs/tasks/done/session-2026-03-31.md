## Implemented

### 1. Batch sub-request error detection (batch.py)
- Added `_check_batch_errors()` helper that inspects sub-response statuses
- `execute_batch` now raises `BatchError` when any sub-request returns
  non-2xx status (was silently ignored)
- Added `raise_on_errors=False` escape hatch for manual error handling
- 6 new tests (partial failure, multi-failure, opt-out, sync + async)

### 2. EventFrames.create endpoint fix (eventframes.py)
- Changed from `POST /elements/{webId}/eventframes` (invalid) to
  `POST /assetdatabases/{webId}/eventframes` (correct)
- Renamed parameter `element_web_id` to `database_web_id`
- Added `ref_element_web_ids` parameter for element association
- Updated all tests to use correct endpoint
- 1 new test for ref_element_web_ids body content

### 3. BasicAuth credential redaction (auth.py)
- Wrapped `httpx.BasicAuth` with `_RedactedAuth` (same as NTLM)
- `repr(basic_auth("user", "pass"))` now shows
  `BasicAuth(credentials=<redacted>)` instead of leaking the password
- Updated existing tests; 1 new test for repr redaction

### 4. base_url scheme validation (client.py)
- Both sync and async clients now reject non-http(s) schemes with
  `ValueError` at construction time
- Previously, `ftp://` or empty scheme would produce an opaque error
  at request time
- 2 new tests (invalid scheme, empty scheme)

### 5. boundaryType parameter (values.py, streamsets.py)
- Added `boundary_type` keyword-only param to:
  - `StreamsMixin.get_recorded` (sync + async)
  - `StreamSetsMixin.get_recorded_ad_hoc` (sync + async)
  - `StreamSetsMixin.get_recorded_by_element` (sync + async)
- Supports "Inside", "Outside", "Interpolated" — omitted by default
- 4 new tests (param forwarding, omission-by-default, values + streamsets)
