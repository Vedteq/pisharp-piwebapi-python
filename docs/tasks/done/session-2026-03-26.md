## Implemented This Session

### 1. Feature: `get_end` on streams mixin
- `GET /streams/{webId}/end` — returns last archived event (distinct from snapshot)
- Sync + async versions
- Happy-path and 404 error-path tests (4 tests)

### 2. Feature: `filterExpression` on `get_recorded`
- Added `filter_expression` keyword-only parameter to `get_recorded()`
- Server-side filtering using PI performance equation syntax
- Omitted from query when `None` (backward compatible)
- Sync + async versions
- Tests for parameter forwarding and omission (3 tests)

### 3. Feature: Element Templates resource
- New `elementtemplates.py` with `ElementTemplatesMixin` + `AsyncElementTemplatesMixin`
- Methods: `get_by_web_id`, `get_by_path`, `get_by_database`, `get_attribute_templates`
- Wired into `PIWebAPIClient` and `AsyncPIWebAPIClient` as `client.elementtemplates`
- 14 tests covering happy paths, error paths (404, 401), and parameter forwarding

### 4. Security Fix: auth_method validation and verify_ssl warning
- Reject unknown `auth_method` values with `ValueError` (prevents silent unauthenticated requests)
- Require `username` + `password` when `auth_method` is `"basic"` or `"ntlm"`
- Emit `UserWarning` when `verify_ssl=False` to alert developers
- Both sync and async clients
- 6 tests for validation and warning behavior

### 5. Bug Fix: EventFrame.end_time now optional
- Changed `end_time: datetime` to `end_time: datetime | None` in `EventFrame` model
- Open (in-progress) event frames return `null` for `EndTime` from PI Web API
- Previously caused Pydantic `ValidationError` when parsing open event frames
- 2 tests added for null and missing EndTime

### 6. Bug Fix: create_point return type corrected
- Changed `create_point()` return type from `PIPoint` to `None`
- PI Web API returns HTTP 201 with empty body (not a JSON PIPoint object)
- Previous implementation would crash with `JSONDecodeError` against a real server
- Updated all create_point tests to expect `None` and mock 201 with empty body
