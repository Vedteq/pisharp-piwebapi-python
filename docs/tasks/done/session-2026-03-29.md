## Gaps Found

**Already implemented:** Points, Streams, StreamSets, Calculation, Search,
Elements (with attributes), EventFrames, ElementTemplates, Servers
(Asset/Data/Databases), Batch, Pagination.

**Key gaps identified:**
1. Attribute value get/set (`GET/PUT /attributes/{webId}/value`) — critical
2. Element update (`PATCH /elements/{webId}`) — missing CRUD operation
3. System/home endpoint (`GET /`) — connectivity check & discovery
4. Table / TableCategory controller — AF tables
5. EnumerationSet / EnumerationValue controller — digital state management
6. SecurityIdentity / SecurityMapping controller — security admin
7. Channel controller — WebSocket real-time streaming
8. NotificationRule controller (model exists, no resource)

## Bugs Found (Code Reviewer)

1. **eventframes.search used non-existent endpoint** — `/eventframes/search`
   does not exist in PI Web API. Fixed to use
   `GET /assetdatabases/{webId}/eventframes`. BREAKING: signature changed.
2. **PIAttribute model missing fields** — TypeQualifier, ConfigString,
   DataReferencePlugIn, HasChildren, IsConfigurationItem were silently dropped.
3. **Double error-check** — event hooks + manual raise_for_response in mixins.
   Noted but not fixed (harmless, deferred to avoid churn).
4. **servers.py duplicates elements.py** — same endpoints exposed via two
   accessors. Consolidation deferred.

## Implemented

### 1. `get_attribute_value` / `set_attribute_value` (elements.py)

- `PUT /attributes/{webId}/value` → accepts scalar or dict body
- Both sync and async versions
- 4 sync tests + 4 async tests (happy path + error path + scalar/dict variants)

### 2. `update_element` (elements.py)
- `PATCH /elements/{webId}` with arbitrary body dict
- Both sync and async versions
- 2 sync tests + 2 async tests (happy path + not found)

### 3. `home()` method (client.py) + `PISystemInfo` model
- `GET /` → returns `PISystemInfo` with product_title, product_version, links
- Both sync and async versions on the client class directly
- 2 sync tests + 2 async tests (happy path + auth failure)
- New `PISystemInfo` model exported from `__init__.py`

- Changed from non-existent `/eventframes/search` to correct
  `GET /assetdatabases/{webId}/eventframes`

- Uses `nameFilter` instead of `q` query param
- Added error-path tests for both sync and async

### 5. Fix: Enriched `PIAttribute` model (models.py)
- Added: `type_qualifier`, `config_string`, `data_reference_plugin`,
  `has_children`, `is_configuration_item`
