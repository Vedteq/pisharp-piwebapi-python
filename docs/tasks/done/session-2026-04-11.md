## What was implemented

### 1. Fix Items fallback bug (`fix:` commit)

Replaced `data.get("Items", data) if isinstance(data, dict) else data`
with `data.get("Items", []) if isinstance(data, dict) else data` across
12 files (42 call sites). When the API returns a dict without an
`Items` key, the previous fallback would iterate the dict's keys as
"items", producing a `ValidationError` on each key. The new fallback
returns an empty list instead.

Added 2 regression tests in `test_coverage_gaps.py`
(`TestItemsFallbackEmptyResponse`) — one for `points.search`, one for
`elements.get_child_elements` — that assert `[]` is returned when the
server responds with `{}`.

Files touched: `analyses.py`, `analysistemplates.py`, `attributes.py`,
`attributetemplates.py`, `elements.py`, `elementtemplates.py`,
`enumerationsets.py`, `notificationrules.py`, `points.py`,
`securityidentities.py`, `securitymappings.py`, `unitclasses.py`.

### 2. NotificationRuleTemplate controller (`notificationruletemplates.py`)

New standalone controller covering the full template lifecycle:

- `get_by_web_id(web_id, selected_fields=...)`
- `get_by_path(path, selected_fields=...)`
- `get_by_database(database_web_id, name_filter=..., max_count=..., selected_fields=...)`
- `create_on_database(database_web_id, body)`
- `get_subscribers(web_id)` — returns `list[PINotificationRuleSubscriber]`
- `update(web_id, body)`
- `delete(web_id)`

Both sync and async mixins; `PINotificationRuleTemplate` Pydantic model
added to `models.py` with fields Name, Description, Criteria,
MultiTriggerEventOption, NonrepetitionInterval, ResendInterval,
TargetName, etc. Wired into both clients as
`client.notificationruletemplates`. Exported from `__init__.py`. 22
new tests (sync + async; happy path, selected_fields, empty-response
regression, NotFound, 500, validation).

### 3. AttributeTrait controller (`attributetraits.py`)

New read-only controller exposing the built-in attribute trait
catalogue:

- `get_by_name(name, selected_fields=...)`
- `get_multiple(names, selected_fields=...)` — repeated `name` params
- `get_categories()` — returns `list[str]`

Both sync and async. `PIAttributeTrait` Pydantic model added with
fields Name, Abbreviation, Description, AllowChildAttributes,
AllowDuplicates, IsDataReferenceInherited. Wired as
`client.attributetraits`. Exported from `__init__.py`. 14 new tests
(sync + async; happy path, selected_fields, flat-list response path,
NotFound, empty-input validation).
