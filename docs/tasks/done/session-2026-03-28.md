## Implemented This Session

### 1. Bug Fix: `extra_fields` override order in `create_point`
- Changed from `body.update(extra_fields)` (extra_fields wins) to building body from extra_fields first then applying standard params on top (standard params win)
- Applied to both sync and async `create_point` methods
- Added test verifying standard params cannot be overridden by extra_fields
- 1 new test

### 2. Feature: Calculation resource (`client.calculation`)
- New `calculation.py` with `CalculationMixin` + `AsyncCalculationMixin`
- Methods:
  - `get_recorded(expression, ...)` — evaluate expression at recorded event times
  - `get_summary(expression, ...)` — compute summary statistics of an expression
  - `get_at_intervals(expression, ..., sample_interval)` — evaluate at regular intervals
  - `get_at_times(expression, times, ...)` — evaluate at specific timestamps
- All methods support optional `web_id` parameter for context (relative tag references like `'.'`)
- Wired into `PIWebAPIClient` and `AsyncPIWebAPIClient` as `client.calculation`
- 17 tests covering happy paths, parameter forwarding, web_id omission, error handling, and all 4 async methods

### 3. Feature: Indexed AF Search (`client.search`)
- New `search.py` with `SearchMixin` + `AsyncSearchMixin`
- `query(q, *, scope, fields, max_count, start_index)` method calling `GET /search/query`
- Supports PI AF indexed search syntax: `name:Pump*`, `templateName:Equipment`, boolean operators
- New `SearchResult` class with `web_id`, `name`, `path`, `item_type`, `score`, `raw` attributes
- `SearchResult` exported from `__init__.py`
- Wired into both client classes as `client.search`
- 12 tests covering happy paths, parameter forwarding, empty results, errors, defaults, and repr

### 4. Security Fix: SSRF protection in pagination
- `get_all_pages` now validates that `Links.Next` URLs share the same origin (scheme + host) as the client's `base_url`
- Prevents a malicious PI Web API server from redirecting the SDK (with auth credentials) to internal network addresses (e.g. cloud metadata endpoints at `169.254.169.254`)
- Both sync and async pagination paths protected
- 3 new tests (sync cross-origin rejection, sync same-origin allow, async cross-origin rejection)

### Bug Fixes (from code review)
9. **`cert + verify_ssl=False` silently enables TLS verification** — `ssl.create_default_context()` overrides `verify_ssl=False` when a cert is provided
10. **`EventFramesMixin.create` requires `end_time`** — should be optional for open (in-progress) event frames, matching the model
11. **`create_element` can't create root-level database elements** — needs `create_database_element` using `POST /assetdatabases/{webId}/elements`
12. **`get_values` (streamsets snapshot) drops identity fields** — returns `list[StreamValue]` losing WebId/Name/Path; should be `list[StreamSetItem]` or document order-dependency
13. **Empty `web_ids=[]` in streamsets sends malformed request** — should raise `ValueError`

### Test Gaps (from code review)
14. **`create_element`, `create_attribute`, `delete_element`, `delete_attribute`** — zero tests
15. **Async `eventframes.create`, `eventframes.delete`, `eventframes.acknowledge`** — no async tests
16. **`BatchError` exported but never raised** — `execute_batch` doesn't inspect sub-response failures

### Security (from security audit)
17. **Warn on HTTP base_url** — credentials sent in plaintext over HTTP without warning
18. **Truncate error message bodies** — prevent unbounded allocation from malicious servers
19. **Sanitize SSL cert loading errors** — private key paths may leak in tracebacks
